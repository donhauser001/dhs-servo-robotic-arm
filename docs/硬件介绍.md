# 舵机机械臂 2 号 — 硬件介绍

> 一台 **6 自由度** 串行总线舵机机械臂。主控 Raspberry Pi 5 + 微雪 Bus Servo Driver HAT (A) + 飞特 ST 系列总线舵机（ST3020 / ST3025 / ST3235）。

---

## 1. 系统总览

```
            ┌──────────────────────┐
            │   Raspberry Pi 5     │  上位机：算法、运动学、Web UI
            │   (Debian / Linux)   │
            └──────────┬───────────┘
                       │  40-pin GPIO 串口（/dev/serial0, 1Mbps）
                       │  或：Type-C USB 直连 PC 调试
            ┌──────────┴───────────┐
            │ Bus Servo Driver HAT │  下位机/总线驱动
            │  (A)  ESP32-WROOM    │  ESP32 透传 / 直通 USB
            └──────────┬───────────┘
                       │  TTL 半双工 1Mbps（飞特 SMS_STS 协议）
                       │  6P 排线菊花链（VCC / GND / Signal）
                       │
   ┌───────────┬───────┴───────┬──────────┬──────────┬──────────┐
   ▼           ▼               ▼          ▼          ▼          ▼
┌────────┐ ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐
│ ST3020 │ │ ST3025 │  │ ST3025 │  │ ST3235 │  │ ST3235 │  │ ST3020 │
│  ID=1  │ │  ID=2  │  │  ID=3  │  │  ID=4  │  │  ID=5  │  │  ID=6  │
│ 底座   │ │ 大臂   │  │ 小臂   │  │ 腕俯仰 │  │ 腕旋转 │  │ 末端   │
│  J1    │ │  J2    │  │  J3    │  │  J4    │  │  J5    │  │  J6    │
└────────┘ └────────┘  └────────┘  └────────┘  └────────┘  └────────┘
```

**核心要点**：

- 所有舵机和驱动板共用一条 **TTL 半双工总线**，只通过 ID 区分。
- 三款舵机虽规格不同，但**协议完全一致**（飞特 SMS_STS / SCS Protocol），软件层无需差异化处理。
- 主控树莓派 5 装在 HAT 上方，HAT 通过 GPIO 串口与树莓派通信，并集成 5V DC-DC 给树莓派供电。

## 2. 关节配置（6 DOF）

| 关节 | 名称 | 功能 | 舵机型号 | 默认 ID | 堵转扭矩 | 备注 |
| :---: | :--- | :--- | :--- | :---: | :---: | :--- |
| **J1** | base | 底座旋转 | ST3020 | 1 | 25 kg·cm | 双轴金属齿轮 |
| **J2** | shoulder | 大臂关节 | **ST3025** | 2 | **40 kg·cm** | 双轴**无刷**钢制齿轮，承受最大重力矩 |
| **J3** | elbow | 小臂关节 | **ST3025** | 3 | **40 kg·cm** | 双轴**无刷**钢制齿轮 |
| **J4** | wrist_pitch | 腕部俯仰 | ST3235 | 4 | 30 kg·cm | 单轴金属齿轮 |
| **J5** | wrist_roll | 腕部旋转 | ST3235 | 5 | 30 kg·cm | 单轴金属齿轮 |
| **J6** | end_roll | 末端旋转 | ST3020 | 6 | 25 kg·cm | 双轴金属齿轮，安装末端工具 |

> **设计逻辑**：
> 越靠近底座的关节负载越大，所以 J2/J3 用扭矩最大的 ST3025（无刷，寿命长）；
> 腕部需要细腻动作，用 ST3235（30 kg·cm）；
> 底座和末端只需中等扭矩，用 ST3020。

## 3. 部件清单

### 3.1 主控：Raspberry Pi 5

| 项目 | 规格 |
| :--- | :--- |
| SoC | Broadcom BCM2712（4× Cortex-A76 @ 2.4 GHz）|
| 内存 | LPDDR4X-4267（4 GB / 8 GB）|
| 串口 | UART0 = `/dev/ttyAMA0`（GPIO14/15），别名 `/dev/serial0` |
| 供电 | 5V/5A（USB-C PD），HAT 自带 5V/3A DC-DC 可代供 |
| OS | Raspberry Pi OS (Debian 12 / Bookworm) |

**GPIO 串口启用方法**（Raspberry Pi OS）：

```bash
sudo raspi-config nonint do_serial_hw 0     # 启用串口硬件
sudo raspi-config nonint do_serial_cons 1   # 关闭串口控制台
echo "dtoverlay=disable-bt" | sudo tee -a /boot/firmware/config.txt   # 释放 UART0
sudo reboot
```

启用后 `/dev/serial0` → `/dev/ttyAMA0`，1Mbps 直接走 GPIO14(TX)/GPIO15(RX)。

### 3.2 总线舵机驱动板：Waveshare Bus Servo Driver HAT (A)

| 项目 | 规格 |
| :--- | :--- |
| 主控芯片 | ESP32-WROOM-32（双核 240 MHz，板载 WiFi）|
| 板载电路 | 总线舵机驱动（TTL 半双工 + RS485）|
| 供电接口 | XT60 + DC 5.5×2.5 mm（**9–25V**，需匹配舵机电压）|
| 通信接口 | 树莓派 40-pin GPIO UART × 1，Type-C USB × 2，TTL/RS485 舵机端口 |
| 输出 | 集成 5V/3A 降压，可经 GPIO 给树莓派供电 |
| PCB | 65 × 57 mm，定位孔 58 × 49 mm |
| 文档 | <https://www.waveshare.net/wiki/Bus_Servo_Driver_HAT_(A)> |

**两种工作档位**（板上拨码切换）：

| 档位 | 数据通路 | 适用场景 |
| :--- | :--- | :--- |
| **USB 档** | PC ↔ Type-C ↔ CH343 ↔ 舵机总线 | **PC 调试**：直接用 Mac/PC 写 Python 控制，最适合开发期 |
| **ESP32 档** | 树莓派 ↔ GPIO UART ↔ ESP32 ↔ 舵机总线 | **整机运行**：ESP32 烧"透传固件"后做 USB-UART 桥接 |

> ESP32 档第一次使用前必须给 ESP32 烧录"舵机透传"固件（驱动板出厂可能不是这套）。
> 烧录工具：[ESP32 Flash Download Tool](https://www.waveshare.net/w/upload/8/8b/SERVO_DRIVER_WITH_ESP32_ST_FACTORY.rar)。

## 4. 通信协议总结

三款舵机和驱动板均使用 **飞特 SMS_STS 协议**：

| 项目 | 取值 |
| :--- | :--- |
| 物理层 | TTL 半双工（共一根信号线收发）|
| 默认波特率 | **1 Mbps**（可配 38400 ~ 1 Mbps）|
| 字节序 | **小端**（low byte first）|
| 帧格式 | `0xFF 0xFF` + ID + LEN + INST + PARAMS + CHKSUM |
| 校验 | `CHKSUM = ~(ID + LEN + INST + PARAMS) & 0xFF` |
| 广播 ID | `0xFE` |
| 位置量纲 | 0 – 4095（对应 360°，分辨率 0.088°）|
| 速度量纲 | 0 – 3073 步/秒（50 步/秒 ≈ 0.732 RPM）|
| 加速度量纲 | 0 – 150（数值越小越柔和）|

**关键指令**：

| 指令 | 字节 | 说明 |
| :--- | :---: | :--- |
| PING | `0x01` | 探测舵机是否在线 |
| READ | `0x02` | 读内存表 |
| WRITE | `0x03` | 写内存表 |
| REG_WRITE | `0x04` | 缓存写（等 ACTION 触发执行）|
| ACTION | `0x05` | 触发所有缓存写一起生效（多机同步起步）|
| SYNC_WRITE | `0x83` | 同步写（一帧给多个舵机各发各的目标）|
| RESET | `0x06` | 恢复出厂 |

**关键内存地址**（节选，详见 [`arm/protocol.py`](../arm/protocol.py) 的 `Reg` 枚举）：

| 地址 | 名称 | 大小 | 区域 | 说明 |
| :---: | :--- | :---: | :--- | :--- |
| 0x05 | ID | 1 B | EPROM | 舵机 ID（出厂默认 1）|
| 0x28 | TORQUE_ENABLE | 1 B | SRAM | 0=放扭矩, 1=锁扭矩 |
| 0x29 | ACC | 1 B | SRAM | 启停加速度 |
| 0x2A | GOAL_POSITION | 2 B | SRAM | 目标位置 0–4095 |
| 0x2E | GOAL_SPEED | 2 B | SRAM | 目标速度 0–3073 |
| 0x37 | LOCK | 1 B | SRAM | EPROM 锁，0=解锁后才能写 EPROM |
| 0x38 | PRESENT_POSITION | 2 B | 反馈 | 当前位置 |
| 0x3A | PRESENT_SPEED | 2 B | 反馈 | 当前速度（bit15=方向）|
| 0x3C | PRESENT_LOAD | 2 B | 反馈 | 当前负载（bit15=方向）|
| 0x3E | PRESENT_VOLTAGE | 1 B | 反馈 | 单位 0.1 V |
| 0x3F | PRESENT_TEMPERATURE | 1 B | 反馈 | ℃ |

## 5. 接线与供电

```
   ┌──────────────────────────────┐
   │       12V 电源（≥5A）         │   建议 12V/10A 开关电源或 3S 锂电
   │                              │   保护：保险丝、共地、避免反接
   └────────────┬─────────────────┘
                │ XT60
                ▼
   ┌──────────────────────────────┐
   │   Bus Servo Driver HAT (A)   │
   │                              │
   │   ┌─ 5V/3A DC-DC ─► 树莓派 5 │  (经 40-pin 给树莓派供电)
   │   │                          │
   │   └─ 12V 直通  ──► 舵机总线   │  (6 颗舵机共享)
   └────────────┬─────────────────┘
                │ TTL 6P 排线
                ▼
       J1 → J2 → J3 → J4 → J5 → J6   (菊花链：每颗舵机两个口，串联即可)
```

**供电要点**：

- **总电流估算**：6 × 堵转电流上限约 4 A → 理论峰值 24 A，但实际**6 颗同时堵转概率极低**。
  建议**12V/10A** 电源，留 30% 余量，配 15A 保险丝。
- **三种供电场景**：
  1. **PC 调试 + 单独 12V 给 HAT**：USB 供数据，DC 供舵机能量。最稳妥。
  2. **整机运行**：12V 进 HAT 的 XT60 → HAT 自降 5V 给树莓派 + 12V 给舵机。线最少。
  3. **树莓派 USB-C 单独供电** + HAT 12V 单独供电：双路独立，电源最干净。
- **务必共地**！如果树莓派和舵机用两套电源，必须把它们的 GND 接在一起。

## 6. 已知风险与防护

| 风险 | 表现 | 防护措施 |
| :--- | :--- | :--- |
| **舵机 ID 冲突** | 通信失败、状态包错误 | 第一次使用必须**逐个**改 ID（接一个改一个）|
| **EPROM 写不进** | 改 ID/标定零位掉电丢失 | 写 EPROM 前必须 `unLockEprom`，写完 `LockEprom` |
| **过流烧机** | 螺丝过长顶到机体、关节卡死 | 严格用配套螺丝；监控 `PRESENT_CURRENT`，超阈值放扭矩 |
| **过热** | 长时间堵转、负载过大 | 监控 `PRESENT_TEMPERATURE`，>65℃ 报警，>70℃ 自动放扭矩 |
| **电压跌落** | 多舵机同时启动瞬时大电流 | 加大电源容量、电源加电解电容、用 `ACC` 设柔启动 |
| **机械干涉** | 关节超限位撞机壳 | 在配置文件设 `min_angle_deg / max_angle_deg`，软件预先夹紧 |
| **断电失控** | 突然掉电时机械臂自由下坠 | 机械结构尽量自重平衡；关节齿轮加阻尼 |

## 7. 软件如何看待硬件

软件层把硬件抽象成三层（详见 `arm/` 目录）：

```
┌─────────────────────────────────────┐
│  arm.Arm        6 DOF 整体控制器     │ ← 应用层（运动学、轨迹）
├─────────────────────────────────────┤
│  arm.Servo      单舵机抽象           │ ← 关节级（角度↔原始位置）
├─────────────────────────────────────┤
│  arm.Bus        半双工串口总线       │ ← 驱动级（收发字节）
├─────────────────────────────────────┤
│  arm.protocol   SMS_STS 协议封装     │ ← 协议级（构造/解析帧）
└─────────────────────────────────────┘
```

**对硬件的核心假设**（写在 `config/arm_config.yaml`）：

- 总线只有一条（一个串口），全部舵机共享。
- 每个关节的舵机原始位置 `0–4095` 对应物理角度由 `home_raw` + `direction` + 关节角度区间映射。
- 标定流程必须先做：把每个关节摆到机械零位 → 用 `tools/calibrate.py` 记下当前 raw 值作为 `home_raw`。
